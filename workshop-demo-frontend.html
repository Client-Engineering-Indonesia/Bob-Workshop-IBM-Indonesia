☕ Java Modernization Demo

Legacy vs Modern Code Comparison

❌ Legacy Java 6/7
// SECURITY RISK: Hardcoded credentials
private static final String DB_PASSWORD = "P@ssw0rd123";

// SQL INJECTION VULNERABILITY
public double getAccountBalance(String accountNumber) {
    Statement stmt = connection.createStatement();
    
    // Concatenating user input - DANGEROUS!
    String query = "SELECT balance FROM accounts " +
                   "WHERE account_number = '" + accountNumber + "'";
    
    ResultSet rs = stmt.executeQuery(query);
    
    if (rs.next()) {
        return rs.getDouble("balance");
    }
    
    // Poor error handling
    catch (SQLException e) {
        e.printStackTrace(); // Exposes internal details!
    }
}
✅ Modern Java 17 + Spring Boot 3
@Service
@Slf4j
@RequiredArgsConstructor
public class BankingService {
    
    private final AccountRepository accountRepository;
    
    // Credentials in environment variables
    // Parameterized queries prevent SQL injection
    public BigDecimal getAccountBalance(@NotNull String accountNumber) {
        log.info("Retrieving balance for account: {}", accountNumber);
        
        return accountRepository.findByAccountNumber(accountNumber)
                .map(Account::getBalance)
                .orElseThrow(() -> 
                    new AccountNotFoundException(accountNumber));
    }
}

✅ Key Improvements

  • Java 17 LTS with latest security patches
  • Spring Boot 3 with dependency injection
  • Parameterized queries prevent SQL injection
  • Proper logging with SLF4J
  • BigDecimal for precise financial calculations

💾 COBOL Modernization Demo

Legacy vs Modern Code Comparison

❌ Legacy - GOTO Spaghetti
MENU-LOOP.
    ACCEPT WS-MENU-CHOICE.
    
    IF WS-MENU-CHOICE = 1
        GO TO CREATE-ACCOUNT
    ELSE IF WS-MENU-CHOICE = 2
        GO TO VIEW-BALANCE
    ELSE IF WS-MENU-CHOICE = 3
        GO TO DEPOSIT-MONEY
    ELSE IF WS-MENU-CHOICE = 4
        GO TO WITHDRAW-MONEY
    ELSE IF WS-MENU-CHOICE = 5
        GO TO TRANSFER-MONEY
    ELSE
        GO TO MENU-LOOP.

* Impossible to follow program flow
* No structure or modularity
* Money can be lost in transfers!
✅ Modern - Structured Programming
0000-MAIN-PROGRAM.
    PERFORM 1000-INITIALIZE-PROGRAM
    PERFORM 2000-PROCESS-MENU UNTIL EXIT-PROCESSING
    PERFORM 9000-TERMINATE-PROGRAM
    STOP RUN.

2000-PROCESS-MENU.
    PERFORM 2100-DISPLAY-MENU
    PERFORM 2200-GET-USER-CHOICE
    
    EVALUATE MENU-CHOICE
        WHEN 1 PERFORM 3000-CREATE-ACCOUNT
        WHEN 2 PERFORM 4000-VIEW-BALANCE
        WHEN 3 PERFORM 5000-DEPOSIT-MONEY
        WHEN 4 PERFORM 6000-WITHDRAW-MONEY
        WHEN 5 PERFORM 7000-TRANSFER-MONEY
        WHEN 9 SET EXIT-PROCESSING TO TRUE
    END-EVALUATE.

✅ Key Improvements

  • Structured programming - NO GOTO statements
  • Transaction management with rollback capability
  • Meaningful variable names
  • Comprehensive error handling with codes
  • Complete audit trail
