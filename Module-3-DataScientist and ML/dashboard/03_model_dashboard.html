Error Analysis

Common Misclassifications

ERROR → MATCHED (25 cases):

Model incorrectly predicts successful settlement for error cases. This is the most critical error type as it may lead to operational issues.

UNMATCHED → MATCHED (41 cases):

Model is overly optimistic, predicting successful settlement for pending cases. May cause false confidence in settlement status.

MATCHED → UNMATCHED (35 cases):

Model is overly cautious, flagging successful settlements as pending. Less critical but may trigger unnecessary reviews.

Model Strengths

  • MATCHED Detection: 75% recall (150/200) - Good at identifying successful settlements
  • UNMATCHED Detection: 58% recall (81/140) - Moderate performance on pending cases
  • Overall Balance: No extreme bias toward any single class

Areas for Improvement

  • ERROR Detection: Only 38% recall (23/60) - Needs improvement for critical error cases
  • Class Imbalance: ERROR class underrepresented in training data
  • Feature Engineering: May need features specifically for error detection

Business Insights from Feature Importance

🔑 Key Findings:

Top 3 Most Important Features

  1. days_to_settle (0.1250): Settlement timeline is the strongest predictor. Longer delays increase risk of errors and unmatched status.
  2. risk_score (0.1100): Our composite risk metric effectively captures settlement risk. Validates the feature engineering approach.
  3. deal_price (0.0950): Trade value significantly impacts settlement outcome. High-value trades require more attention.

Risk Factors Validated

  • OTC Trades (0.0750): Over-the-counter trades are indeed higher risk, as suspected by domain experts
  • Major Currencies (0.0700): Currency type matters - major currencies (USD/EUR/GBP/JPY) have better settlement rates
  • Priority Levels (0.0600): Settlement priority is a significant factor in outcomes

Actionable Recommendations

  • Monitor trades with days_to_settle > 3 more closely
  • Flag trades with risk_score >= 2 for manual review
  • Prioritize high-value OTC trades in non-major currencies
  • Implement automated alerts for high-risk combinations
